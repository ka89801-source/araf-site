# ARAF PLATFORM – AI AGENT RULES

This repository contains the codebase for the Araf legal platform.

AI agents working on this repository must follow the rules below.

---

## PRIMARY RULE

Do NOT make destructive changes.

Never remove or rewrite core functionality unless explicitly instructed.

Always prefer minimal and safe modifications.

---

## PROJECT IDENTITY

Araf is a Saudi legal platform.

The interface must remain:

• Arabic
• RTL layout
• Professional and elegant
• Simple and fast

Never convert the project to LTR.

Never remove Arabic text.

---

## CRITICAL AREAS – DO NOT BREAK

The following must remain stable:

• Homepage structure
• Search functionality
• Contact form
• Deployment compatibility with Vercel
• Mobile responsiveness

If modifications affect these areas, verify functionality after changes.

---

## DEVELOPMENT RULES

When modifying code:

1. Make incremental changes only.
2. Avoid large refactors unless necessary.
3. Reuse existing components whenever possible.
4. Do not change project structure without justification.

---

## SAFETY CHECK

Before completing any task:

• Verify build success
• Ensure no UI layout break
• Ensure Arabic text rendering remains correct
• Ensure mobile layout remains functional

---

## UI PRINCIPLES

The interface should feel:

• modern
• premium
• minimal
• fast

Avoid heavy animations or unnecessary libraries.

---

## TASK EXECUTION

For any requested change:

1. Inspect the repository first
2. Understand the existing implementation
3. Apply minimal improvements
4. Validate build and UI integrity
5. Summarize changes clearly
